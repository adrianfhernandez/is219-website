02

Class Project

Stock database with hourly market updates and server integration.

This class project created a stock information database with a complete schema, field definitions, and column structure. I coded the data pipeline to transfer stock trade data into the project’s main server and website, then configured it to refresh hourly with live market updates.

SQL Schema Design Database Engineering ETL / Hourly Sync Server Integration Web Data Display

Step 1: Define the Data Model

Designed the database schema for stock symbols, trade prices, timestamps, volumes, and market metadata. Each field and column was carefully structured for fast updates and reporting.

Step 2: Build the Transfer Pipeline

Implemented the code that moves stock data into the project’s main server and database. This ensured the website could access the stock dataset reliably and securely.

Step 3: Enable Hourly Updates

Configured the system for hourly refreshes so the stock database stays current with actual market trades. The scheduled sync keeps the site portfolio data aligned with live trading activity.

Step 4: Document the Architecture

Provided server documentation explaining the database design, field structure, and transfer process. This documentation describes how the database supports the main project website and server workflow.
